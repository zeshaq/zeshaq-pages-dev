This section is the documentation hub for the full-platform rebuild.

The project scope covers the OpenShift fleet, disconnected image supply,
GitOps operating model, platform services, compliance guardrails, and the
runbooks needed to repeat the deployment in another environment.

## Current baseline

- Management cluster: `hub-dc-v6`
- Workload cluster: `spoke-dc-v6`
- Base domain: `sub.comptech-lab.com`
- Platform source of truth: GitOps-managed OpenShift desired state
- Runtime posture: disconnected-first, with local mirrors for OpenShift image
  supply

## Documentation tracks

- [Full platform overview](/blog/full-platform-overview)
- Architecture decisions and operating model
- Disconnected install and mirror workflow
- OpenShift GitOps and RHACM fleet management
- Operator installation and lifecycle
- Vault, Nexus, MinIO, DNS, ingress, and supporting services
- Compliance readiness and PCI DSS evidence
- Admin handoff and break-glass procedures

## Rule for this section

Content here should describe the desired operating model and repeatable
procedures. Temporary notes, one-off troubleshooting, and secrets should stay
out of the public blog.
