> Built 2026-05-11 for the BRAC engagement. The [scope email](/brac-poc/demo-scope-email)
> asks for an end-to-end story across IAM, API exposure, mTLS and
> canary. This app delivers the IAM leg with two domain services,
> two SPAs, and a Let's Encrypt cert — all the runtime config flows
> through GitOps.

## What got built

A payment workflow with split duty between **customers** (initiate
transfers) and an **approver** (settles them):

| App | Role |
| --- | --- |
| `payment-client-svc` | Customer API + system of record for accounts and transfers (Open Liberty 26, MP-JWT 2.1). |
| `payment-approver-svc` | Admin-only API. Forwards the admin's bearer down to client-svc, so the audit trail captures the actual deciding human. |
| `client-frontend` | React SPA for `zahid` / `shaikat` — balance, send-payment, history. |
| `approver-frontend` | React SPA for `admin` — pending queue, approve / decline, audit. |

Identity is one WSO2 IS tenant (`bank-payment`) with three users
sharing one password. The two SPAs are separate OIDC service
providers; tokens are issued via authorization-code + PKCE and
consumed by the backends as id_tokens.

## How it was built

Top-to-bottom on **spoke-dc-v6** in one afternoon, in this order:

1. **Tenant scaffolded via platform-gitops** — Namespace, RBAC,
   LimitRange, ResourceQuota, default-deny NetworkPolicy, Argo
   `Application` pointed at the new app repo's `manifests/base/`.
2. **WSO2 IS tenant bootstrapped by REST API** — `bank-payment`
   tenant, three users, two OIDC SPs with redirect-URI regex for
   prod and localhost, all from JSON specs checked into the repo.
3. **Two Liberty services + two SPAs** — JAX-RS resources, the
   `<mpJwt>` block tuned to WSO2's claim shape, CORS configured per
   SPA origin, role derivation that grants `admin` to the username
   `admin` since WSO2's tokens carry no `groups` claim by default.
4. **GitOps reconciles** — every commit lands on the cluster
   through Argo with `selfHeal=true`. No `oc apply` against the
   cluster during the build; even the wildcard cert flows through
   cert-manager + the platform-gitops repo.
5. **Browser-trusted TLS** — cert-manager issues a Let's Encrypt
   wildcard for `*.apps.spoke-dc-v6.sub.comptech-lab.com` via DNS-01
   over RFC2136 against the lab PowerDNS. The spoke
   IngressController's `defaultCertificate` consumes the Secret,
   so every Route is trusted in browsers automatically.

The image supply chain is the only out-of-GitOps step: `buildah`
builds four images locally and pushes to Nexus
(`app-registry.apps.sub.comptech-lab.com/bank-payment/<name>:latest`);
Argo's reconcile loop then rolls out the workloads.

## What's next

- **Persistence** — JDBC + PostgreSQL or a PVC-backed JSON snapshot;
  the model is already split `Account` / `Transfer`, the swap is
  mechanical.
- **API Gateway** — put WSO2 APIM in front of both backends; the
  OpenAPI specs in `wso2/api-definitions/` already carry the
  `x-wso2-*` annotations APIM expects.
- **Service Mesh + mTLS** — add the namespace to the mesh and a
  VirtualService for the canary story; no app code changes.
- **Audit pipeline** — emit a Kafka event per approver decision for
  downstream reconciliation.
