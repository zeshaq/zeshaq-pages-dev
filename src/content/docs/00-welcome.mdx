This documentation captures the architecture, operating model, and runbooks for the CompTech full-platform OpenShift rebuild — a disconnected v6 fleet (`hub-dc-v6` + `spoke-dc-v6`) running ACM, OpenShift GitOps (pull-mode), VM-hosted Vault, ESO, RHACS, OADP, and the federated GitLab / dual-CI build paths used to deliver applications.

## Reading order

The sidebar groups pages roughly in the order operators encounter them:

1. **Architecture** — overview, cluster topology, GitOps operating model.
2. **Image supply** — disconnected mirror, operator lifecycle, Nexus app images.
3. **Storage** — LVMS on the hub, ODF/Ceph on the spoke.
4. **Secrets** — VM Vault, ESO patterns.
5. **Security** — RHACS, image-registry policy.
6. **Backup (OADP)** — DataProtectionApplication wired through Vault credentials.
7. **Compliance** — PCI-DSS readiness via the Compliance Operator.
8. **Operations** — admin handoff, runbooks, known gotchas.
9. **Developer** — onboarding, the two CI paths, app contract.
