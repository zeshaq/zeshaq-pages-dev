This documentation captures the architecture, operating model, and runbooks for the CompTech full-platform OpenShift rebuild — a disconnected v6 fleet (`hub-dc-v6` + `spoke-dc-v6`) running ACM, OpenShift GitOps (pull-mode), VM-hosted Vault, ESO, RHACS, OADP, and the federated GitLab + dual-CI build paths used to deliver applications.

The sidebar groups pages by section. Each leaf page describes one concrete topic. Pages that describe live operational state carry a `Last reviewed` line.

## Reading order

1. **Foundations** — naming, network map, credential rules, glossary.
2. **Lab Infrastructure** — every supporting VM and service outside OpenShift.
3. **OpenShift Platform** — the fleet itself.
4. **Application Delivery** — federated GitLab + the two CI build paths.
5. **Operations** — admin handoff, runbooks, known gotchas.
6. **Architecture Decisions** — ADRs.
7. **References** — quick-lookup tables.
8. **History and Replay** — how we got here, what was tried, and the site-replication plan.
