ACI Simulator is valuable, but only if you use it for the right job.

## What it teaches well

Use it for:

- APIC GUI navigation.
- Tenant and application policy modeling.
- Contracts, filters, and EPG relationships.
- Access policy object relationships.
- L3Out policy structure.
- Faults, health, inventory, audit logs, and object tree inspection.
- API and automation practice.

## What it does not replace

Do not rely on it for:

- Real endpoint traffic.
- Packet captures.
- Hardware forwarding behavior.
- Real BGP or OSPF adjacency troubleshooting.
- Real firewall insertion behavior.
- Real VXLAN EVPN interoperability.
- Performance, scale, or failure testing.

For those, use a separate real-traffic lab later. They are intentionally not
part of this ACI Simulator roadmap.

## Best simulator-only next labs

Continue with APIC policy labs:

| Lab | Why it matters |
| --- | --- |
| Cross-tenant shared services | Learn how to share common services without destroying tenant isolation |
| Common tenant reusable policy | Learn when reusable common filters and contracts help or create risk |
| Contract scope comparison | Understand how broad or narrow contract scope changes blast radius |
| Tenant-specific L3Out models | Compare dedicated external policy per tenant |
| Shared L3Out governance | Model controlled shared external access |
| Service graph concept model | Learn service insertion objects without depending on real packet flow |
| Fault-driven troubleshooting | Learn APIC faults, object relationships, and safe repair workflow |
| Policy export and backup | Practice policy recovery and clean rebuild thinking |

## A practical learning sequence

1. Learn ACI objects in the simulator.
2. Build tenant policy in small labs.
3. Add shared services only after tenant isolation is clear.
4. Practice broad vs narrow contract scope.
5. Add external policy models.
6. Practice faults, audit, backup, and clean replay.
7. Finish with a capstone APIC policy model.

The roadmap page defines the full simulator-only sequence from Lab 1 through
Lab 20.
